# Тесты

Каталог содержит небольшие unit-тесты для основных учебных модулей проекта.
Они нужны, чтобы изменения в формулах метрик, разбиении данных и рекомендательных
моделях не ломали уже объяснённое в учебнике поведение.

Тесты используют маленькие искусственные наборы данных. Благодаря этому ожидаемый
результат можно проверить вручную, а ошибки не скрываются за большим датасетом.

## Что проверяется

### `test_metrics.py`

Проверяет корректность ranking-метрик на tiny-fixture:

- `Precision@K`;
- `Recall@K`;
- `HitRate@K`;
- `MAP@K` через `Average Precision@K`;
- `DCG@K` и `NDCG@K`;
- обработку пустых списков и некорректного `K`;
- группировку рекомендаций по пользователям и порядок по колонке `rank`.

### `test_split.py`

Проверяет time-aware разбиение interactions:

- последнее взаимодействие пользователя попадает в test;
- два последних взаимодействия попадают в validation и test;
- пользователи с недостаточным числом взаимодействий исключаются;
- train не содержит событий из будущего относительно test;
- обязательные колонки и минимальная активность валидируются.

### `test_popularity.py`

Проверяет popularity baseline:

- объекты ранжируются по числу взаимодействий;
- уже просмотренные объекты исключаются;
- рекомендации не содержат дублей;
- для неизвестного пользователя работает глобальный popularity fallback;
- explicit ratings корректно фильтруются по порогу;
- корректно строится множество seen items.

### `test_content_based.py`

Проверяет content-based модель:

- жанры преобразуются в текстовые признаки;
- наиболее похожий объект получает более высокий rank;
- исходный и уже просмотренные объекты не рекомендуются;
- рекомендации не содержат дублей;
- неизвестная история возвращает пустой список.

### `test_item_item.py`

Проверяет item-item collaborative filtering:

- дубли interactions не ломают матрицу;
- сохраняется корректное отображение `item_id` во внутренние индексы и обратно;
- более сильное совместное взаимодействие даёт большую similarity;
- seen items исключаются и рекомендации не дублируются;
- неизвестная история возвращает пустой список.

## Запуск

Для запуска тестов используется Python 3.12.

Проверьте версию активного интерпретатора:

```bash
python --version
```

Ожидаемая версия:

```text
Python 3.12.x
```

Из корня репозитория `recommender-systems-from-zero`:

```bash
PYTHONPATH=src python -m unittest discover -s tests -v
```

Если текущий каталог уже `tests`:

```bash
PYTHONPATH=../src python -m unittest discover -s . -v
```

Ожидаемый результат: все тесты завершаются со статусом `OK`.
